# Formatting Conventions — llm-first-reporting

> Load this file during Phase 0 of the reporting pipeline. These rules govern all report output.

## Core Principle

**Write for humans. Tag for agents.**

Reports generated by `/llm-first-reporting` prioritize readability. The structured project files (ENTITIES.md, ACTIVITIES.md, etc.) are the LLM-optimized source of truth. Reports are projections — they distill that truth into prose that humans can read, share, and annotate.

## Rules

### 1. Prose First

Write in natural language. Use complete sentences and paragraphs. Do not reproduce data tables or entity lists from the structured files — summarize, synthesize, and narrate.

**Bad:**
```
| ACT-023 | Migrate Service Tree data | @deanw | In Progress | Due: 2026-04-01 |
| ACT-024 | Validate mapping rules | @juans | Blocked | Waiting on ACT-023 |
```

**Good:**
```
The Service Tree data migration [ACT-023] is in progress with Dean leading the effort,
targeting an April 1 completion. Mapping rule validation [ACT-024] is blocked until
the migration completes.
```

### 2. Bracket Tags — Sparse, Not Exhaustive

Use `[ACT-023]` bracket tags only when:
- The item is **actionable** — someone reading might want to comment on or follow up
- The item is **specific** — a particular decision, activity, or entity being discussed
- The item is **traceable** — feeding back to the agent with context would be useful

Do NOT tag:
- Every entity mentioned in passing
- Definitions that are obvious from context
- Items listed only for background

**Too many tags:**
```
The ServiceTree [ENT-001] migration [ACT-023] was decided [DEC-007] by the
governance team [ENT-012] following the March review [SRC-003].
```

**Right density:**
```
The ServiceTree migration [ACT-023] was decided by the governance team following
the March review. Dean is leading the effort with an April 1 target.
```

### 3. No Inline Markdown Links

Never use `[ACT-023](ACTIVITIES.md#ACT-023)` in report body. Bracket tags are plain text — they are lightweight markers, not navigable links. The agent knows how to resolve `[ACT-023]` without the full path.

### 4. Headers and Structure

Use clear, descriptive headers that communicate content — not file-internal terminology.

**Bad:** `## ACT-* Status Summary`
**Good:** `## What's In Progress`

Prefer narrative structure:
- **Status Brief:** Summary → What's Working → What's Blocked → What Needs Attention → Next Steps
- **Executive Summary:** Context → Key Findings → Recommendations → Timeline
- **Team Briefing:** Your Focus → Dependencies → Blockers → Decisions Needed
- **Progress Report:** Since Last Report → Completed → In Progress → New Blockers → Upcoming
- **Audit Summary:** Overall Score → Strengths → Vulnerabilities → Top Recommendations

### 5. Audience-Appropriate Language

Adjust language, detail level, and emphasis based on the stated audience:
- **Executives** — outcome-focused, no jargon, 1-page target
- **Team leads** — actionable detail, dependencies highlighted, owners named
- **External auditors** — formal, evidence-cited, methodology references
- **Project managers** — status-focused, timeline-aware, risk-highlighted

### 6. Round-Trip Intent

When writing, imagine the reader will:
1. Read the report
2. Scribble notes next to specific items (e.g., "push this to next sprint", "need to discuss with Dean")
3. Hand the annotated report back to an agent

The bracket tags are anchors for that hand-off. Place them where a reader is most likely to react.

## Tag Resolution Reference

When a report is fed back to an agent, the agent resolves bracket tags using this mapping:

| Tag Pattern | Source File | Example |
|-------------|------------|---------|
| `[ACT-*]` | `ACTIVITIES.md` | `[ACT-023]` → Activity entry with owner, status, evidence |
| `[DEC-*]` | `DECISIONS.md` | `[DEC-007]` → Decision with rationale, participants, date |
| `[ENT-*]` | `ENTITIES.md` | `[ENT-001]` → Entity with role, relationships |
| `[DEF-*]` | `GLOSSARY.md` | `[DEF-012]` → Canonical definition |
| `[SRC-*]` | `SOURCES.md` | `[SRC-003]` → Source with retrieval metadata |
| `[CON-*]` | `CONSISTENCY_REPORT.md` | `[CON-001]` → Conflict with affected sources |
| `[AMB-*]` | `CONSISTENCY_REPORT.md` | `[AMB-002]` → Ambiguity with resolution status |
| `[GAP-*]` | `CONSISTENCY_REPORT.md` | `[GAP-003]` → Gap with suggested resolution |
